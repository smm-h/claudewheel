# Adopt selfdoc root file templates

selfdoc now supports generating root-level files (CLAUDE.md, README.md) from
templates in `docs/` that can contain `:::directive` blocks. Templates are
underscore-prefixed (`docs/_CLAUDE.md`, `docs/_README.md`), and `selfdoc gen`
resolves directives and writes the output to the project root with a
read-only auto-generated header.

## What could use directives

### CLAUDE.md
- **Architecture section** (lines 19-29): replace with `:::list-modules claudewheel` to auto-extract module descriptions from docstrings
- **Config system** (lines 32-36): could use `:::ref claudewheel/config.py` for the migration mechanism docs

### README.md
- **Segment types table** (lines 48-58): replace with `:::table-schema` extracting from segment definitions
- **CLI flags section** (lines 62-106): replace one-shot commands with `:::code-help claudewheel` for auto-extracted help text
- **Config directory table** (lines 112-121): could use `:::table-config` if config structure is extractable
- **Theme overflow table** (lines 152-159): replace with `:::table-schema` from ThemeColors

## Migration steps

1. Create `docs/_CLAUDE.md` -- copy current CLAUDE.md, replace extractable sections with directives
2. Create `docs/_README.md` -- copy current README.md, replace extractable sections with directives
3. Add `"root_files": ["docs/_CLAUDE.md", "docs/_README.md"]` to `selfdoc.json`
4. Add the auto-generated header to existing CLAUDE.md and README.md (first line: `<!-- Auto-generated by selfdoc from docs/_CLAUDE.md -- do not edit -->`)
5. Run `selfdoc gen` to verify output matches expectations
6. Commit the templates and updated root files
